# qa_python_Sprint_4

- Тестирование системы учёта книг (BooksCollector)
Документ описывает набор тестов, реализованных для проверки функциональности класса BooksCollector. Тесты покрывают основные сценарии работы с коллекцией книг: добавление, управление жанрами, работу с избранным и фильтрацию.

- Список реализованных тестов
1. Управление добавлением книг
test_add_new_book_valid_names — проверяет добавление книг с валидными именами (длина ≤ 40 символов).
test_add_new_book_invalid_names — тестирует поведение при попытке добавить книги с невалидными именами:
длина > 40 символов;
пустая строка;
строка из пробелов.
test_add_new_book_duplicate — проверяет, что дубликаты книг не добавляются в коллекцию (размер коллекции остаётся прежним).

2. Управление жанрами книг
test_set_book_genre_valid_case — проверяет успешную установку жанра для существующей книги.
test_set_book_genre_book_not_found — тестирует попытку установить жанр для книги, отсутствующей в коллекции (ожидается None).
test_set_book_genre_invalid_genre — проверяет поведение при установке недопустимого жанра (жанр не меняется, остаётся пустой строкой).
test_get_book_genre_existing_book_with_genre — подтверждает, что метод возвращает установленный жанр для книги с указанным жанром.
test_get_book_genre_existing_book_without_genre — проверяет возврат пустой строки для книги без установленного жанра.
test_get_book_genre_nonexistent_book — тестирует запрос жанра для несуществующей книги (ожидается None).
test_get_books_with_specific_genre — проверяет фильтрацию книг по указанному жанру (включая случай с несуществующим жанром — пустой список).
test_get_books_genre — тестирует полный словарь всех книг и их жанров (с установленными и пустыми жанрами).
test_get_books_for_children — проверяет фильтрацию книг, подходящих для детской аудитории (исключает книги с жанрами типа «Ужасы»).

3. Управление избранными книгами
test_add_book_in_favorites_valid_book — проверяет добавление книги в избранное (книга должна присутствовать в списке избранного).
test_add_book_in_favorites_book_not_in_collection — тестирует попытку добавить в избранное книгу, отсутствующую в коллекции (не должна добавляться).
test_add_book_in_favorites_duplicate — проверяет, что дублирующее добавление книги в избранное не увеличивает размер списка.
test_delete_book_from_favorites — подтверждает удаление книги из избранного (книга отсутствует в списке после удаления).
test_delete_book_from_favorites_book_not_in_favorites — тестирует удаление книги, которой нет в избранном (список остаётся неизменным).
test_get_list_of_favorites_books — проверяет корректность получения полного списка избранных книг (все добавленные книги присутствуют, размер списка соответствует ожиданиям).

- Покрытие функциональности
Тесты покрывают следующие ключевые аспекты системы:

'Валидация данных:
проверка длины имени книги;
обработка невалидных имён (пустые строки, пробелы);
контроль дубликатов.

'Управление жанрами:
установка жанров для существующих книг;
фильтрация по жанрам;
обработка недопустимых жанров;
работа с книгами без жанров.

'Работа с избранным:
добавление/удаление книг из избранного;
предотвращение дублирования в избранном;
получение полного списка избранного.

'Фильтрация и поиск:
поиск книг по жанру;
отбор книг для детской аудитории.

- Используемые инструменты
pytest — фреймворк для запуска тестов и параметризации сценариев.
fixtures — для инициализации объекта BooksCollector перед каждым тестом.
parametrize — для запуска одного теста с разными входными данными (например, валидные/невалидные имена книг, разные жанры).
